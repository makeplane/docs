---
title: Properties
description: A breakdown of all the properties available in issues for members to update
---
| Property       | Description                                                                                                                  |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Title          | Title of the issue or a summary of the issue. This is the only mandatory field to create an issue.                          |
| Description    | A brief about the issue which can contain more information, detailed steps etc.                                             |
| State          | State of the issue in the workflow of the project. When no state is added, the issue is created in the default backlog state. |
| Priority       | Priority of the issue when compared to other issues in the project. Priority can be urgent, high, medium, low, or none.     |
| Assignees      | Members responsible to work on the issue.                                                                                    |
| Labels         | Categorize your issue with labels created within the project.                                                                |
| Start date     | Date when the assignee will start working on the issue.                                                                      |
| Due date       | Date when the assignee is supposed to close the issue.                                                                       |
| Cycle          | If cycles are enabled in your project, you can select the cycle you want the issue to be added in.                          |
| Modules        | If modules are enabled in your project, you can select the modules you want to link to the issue.                           |
| Links          | External links for the issue.                                                                                                |
| Parent         | Mark a parent issue for the issue you are creating.                                                                          |
| Blocking       | Can be linked to issues which are being blocked by the current issue.                                                        |
| Blocked by     | Can be linked to issues which block the current issue.                                                                       |
| Relates to     | Can be linked to issues which are closely related to the current issue.                                                      |
| Duplicate of   | Can be linked to issues which are duplicates of the current issue.                                                           |
