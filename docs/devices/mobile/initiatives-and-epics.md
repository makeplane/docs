# Initiatives & Epics

Initiatives and epics help you plan above the level of a single work item. Initiatives group projects and epics toward a larger goal, while epics group work items inside a project.

## Initiatives <Badge type="info" text="Pro" />

An initiative tracks a major goal across projects and epics. Open **Initiatives** from your workspace to see them, grouped and filterable. To learn the concept, see [Initiatives](/core-concepts/projects/initiatives) on the web.

### Create and edit

- **Create an initiative** with a name and description.
- Set an **icon**, a **lead**, and **labels**.
- Set a **state**: Draft, Planned, Active, Completed, or Closed.
- Set a **start date** and **due date**.

### Scope

The **Scope** of an initiative is the work it covers.

- Add **projects** and **epics** to the scope.
- See a **scope breakdown** of the projects, epics, and work items inside.
- Follow overall **progress** as the scoped work moves forward.

### Details

- Add **links** and **attachments**, and download an attachment when you need it.
- Add **comments** to discuss the initiative.
- Open **Activity** to see the full history of changes.

### Organize the list

- **Filter** by created by, lead, labels, and start or due date.
- **Order by** last created or last updated.
- Use **Display** and **Properties** to control what shows in the list.

## Epics <Badge type="info" text="Pro" />

An epic is a large work item that spans many smaller work items inside a project. Open **Epics** in a project to see them with a progress bar. To learn the concept, see [Epics](/core-concepts/issues/epics) on the web.

- **Create an epic** and set its properties, the same way you would for a [work item](/devices/mobile/work-items).
- Follow each epic's **progress** as its work items are completed.
- **Filter** epics and open their **activity**.
- Add an epic to an initiative's **scope** to roll it up into a larger goal.

::: info
Epics are available from self-hosted `v1.9.0`, and creating an epic from mobile is available from `v1.11.0`.
:::
